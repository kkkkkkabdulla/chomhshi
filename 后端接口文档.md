下面给你一份可直接对接前端的后端接口文档（当前项目实现版）。
默认基础地址：

baseUrl = http://localhost:8080/ssm
统一返回格式：

{
  "code": 200,
  "msg": "success",
  "data": {}
}
鉴权说明：

用户接口：Header 带 token
管理员接口：Header 带 token（管理员登录返回的 token）
公开接口不需要 token 我会单独标注
1) 用户认证与资料
1.1 微信登录（当前为 mock）
POST /api/user/wxLogin
鉴权：否
Body(JSON)
{
  "code": "123456",
  "encryptedData": "mock",
  "iv": "mock"
}
成功返回
{
  "code": 200,
  "msg": "登录成功",
  "data": {
    "token": "xxxxx",
    "user": {
      "id": 1,
      "phone": "13xxxxxx",
      "nickname": "用户0001",
      "avatarUrl": "https://..."
    }
  }
}
1.2 验证 token
GET /api/user/verifyToken
鉴权：是（用户 token）
1.3 获取用户信息
GET /api/user/info
鉴权：是（用户 token）
1.4 更新用户信息
PUT /api/user/updateInfo
鉴权：是（用户 token）
Body(JSON)
{
  "nickname": "新昵称",
  "avatarUrl": "https://example.com/a.png"
}
2) 帖子模块
2.1 发布帖子
POST /api/post/publish
鉴权：是（用户 token）
Body(JSON)
{
  "type": 2,
  "title": "转让高数教材",
  "description": "九成新",
  "category": "书籍",
  "price": 20,
  "location": "",
  "lostFoundTime": "2026-04-05 10:00:00",
  "lostStatus": null,
  "images": "[\"http://.../1.jpg\"]",
  "contact": "vx_123"
}
说明：

type=1（失物招领）时：location/lostStatus 必填
type=2（二手）时：price/category 必填
发布后状态默认 0（待审核）
2.2 我的发布列表
GET /api/post/myList?page=1&pageSize=10
鉴权：是（用户 token）
2.3 公共帖子列表（仅审核通过）
GET /api/post/list?type=2&keyword=教材&category=书籍&page=1&pageSize=10
鉴权：否（公开）
2.4 帖子详情（仅审核通过）
GET /api/post/detail/{id}
鉴权：否（公开）
2.5 编辑帖子
PUT /api/post/update/{id}
鉴权：是（用户 token）
Body：同发布
规则：仅本人 + 仅待审核状态可编辑
2.6 删除帖子
DELETE /api/post/delete/{id}
鉴权：是（用户 token）
规则：仅本人；当前实现中审核通过帖子不允许删除
3) 点赞模块
3.1 点赞/取消点赞（切换）
POST /api/post/like/{id}
鉴权：是（用户 token）
返回
{
  "code": 200,
  "msg": "success",
  "data": {
    "liked": true,
    "likeCount": 12
  }
}
3.2 是否已点赞
GET /api/post/isLiked/{id}
鉴权：是（用户 token）
返回
{
  "code": 200,
  "msg": "success",
  "data": {
    "liked": true
  }
}
4) 评论模块
4.1 新增评论
POST /api/comment/add
鉴权：是（用户 token）
Body(JSON)
{
  "postId": 1,
  "content": "这个还在吗？",
  "parentId": 0
}
4.2 评论列表
GET /api/comment/list/{postId}?page=1&pageSize=10
鉴权：否（公开）
4.3 删除评论
DELETE /api/comment/delete/{id}
鉴权：是（用户 token）
规则：仅评论作者本人
5) 举报模块
5.1 举报帖子
POST /api/report/add
鉴权：是（用户 token）
Body(JSON)
{
  "postId": 1,
  "reason": "虚假信息",
  "description": "描述可选"
}
说明：

举报计数累计到 5 次会自动下架 status=3
返回 msg 区分普通举报成功和自动下架
6) 上传模块
6.1 上传图片
POST /api/upload/image
鉴权：是（用户 token）
Content-Type：multipart/form-data
表单字段：file（文件）
返回：

{
  "code": 200,
  "msg": "success",
  "data": {
    "url": "http://localhost:8080/ssm/api/upload/file/xxxx.jpg"
  }
}
6.2 访问图片文件
GET /api/upload/file/{filename}
鉴权：否（公开）
7) 管理员接口
7.1 管理员登录
POST /api/admin/login
鉴权：否
Body(JSON)
{
  "username": "admin",
  "password": "admin123"
}
7.2 管理员信息
GET /api/admin/profile
鉴权：是（管理员 token）
7.3 帖子审核列表
GET /api/admin/post/list?status=0&page=1&pageSize=10
鉴权：是（管理员 token）
7.4 审核通过
POST /api/admin/post/approve/{id}
鉴权：是（管理员 token）
7.5 审核拒绝
POST /api/admin/post/reject/{id}
鉴权：是（管理员 token）
Body(JSON)
{
  "reason": "内容不规范"
}
7.6 删除帖子
DELETE /api/admin/post/delete/{id}
鉴权：是（管理员 token）
7.7 举报列表
GET /api/admin/post/reportList?status=0&page=1&pageSize=10
鉴权：是（管理员 token）
7.8 处理举报
POST /api/admin/post/handleReport/{reportId}
鉴权：是（管理员 token）
Body(JSON)
{
  "action": "通过",
  "reason": "核实成立"
}
action 当前支持：通过 / 拒绝

7.9 恢复帖子
POST /api/admin/post/restore/{postId}
鉴权：是（管理员 token）
前端调用建议（很关键）
登录后统一把 token 存本地（localStorage / 小程序 storage）
请求封装统一加 header：
用户端：token = userToken
管理端：token = adminToken
遇到 code=401 统一跳登录
上传接口用 multipart/form-data，不要 JSON
图片 URL 直接用返回的 data.url
