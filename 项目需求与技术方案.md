# 校园二手物品与失物招领平台 - 项目需求与技术方案

## 一、项目概述

### 1.1 项目名称
**基于微信小程序的校园二手物品与失物招领平台（SSM + JSP + 微信小程序）**

### 1.2 项目背景
为在校学生提供一个集中、便捷的二手物品交易与失物招领信息发布平台，通过微信小程序的形式方便学生随时随地发布和查询信息。

---

## 二、系统架构与技术选型

### 2.1 整体架构
采用 **前后端分离** 架构：
- **前端**：微信小程序（用户端）+ JSP + Layui 管理后台（管理员端）
- **后端**：SSM（Spring + SpringMVC + MyBatis）+ MySQL
- **通信方式**：RESTful API，统一使用 JSON 数据格式
- **内容审核**：发布前算法检测 + 人工审核 + 用户举报机制

### 2.2 技术栈明细

#### 后端技术栈（已确定）
- **框架**：Spring 5.x + SpringMVC 5.x + MyBatis 3.x
- **数据库**：MySQL 8.0+
- **构建工具**：Maven
- **连接池**：Druid / HikariCP
- **可选增强**：MyBatis-Plus、Lombok
- **接口规范**：RESTful API，统一返回 JSON 格式

#### 前端技术栈
- **微信小程序端（用户端）**
  - 技术：原生微信小程序开发
  - 语言：WXML + WXSS + JavaScript
  - 功能模块：
    - 用户认证：微信一键登录/注册（获取手机号）
    - 帖子管理：发布、浏览、搜索、编辑、删除
    - 互动功能：点赞、评论（查看评论列表）
    - 举报功能：每个帖子角落设置举报按钮
    - 个人中心：查看我的发布、修改昵称和头像
  - 登录状态：首次注册登录，后续自动保持登录（使用本地存储 + token 机制）

- **后台管理端（管理员端）**
  - **技术方案**：JSP + Layui
    - 优势：学习成本低、上手快、不需要 Node.js 环境
    - 适合：时间紧、更熟悉 JSP
  - 功能模块：
    - 管理员登录
    - 帖子审核管理（通过/拒绝/删除）
    - 举报处理（查看举报列表，人工审核）
    - 用户管理（查看、封禁/解封）
    - 数据统计（用户数、帖子数、举报数等）

#### 内容审核技术栈
- **算法检测**：发布前对标题、描述进行敏感词过滤
  - 实现方式：敏感词库匹配（可扩展为更复杂的 NLP 算法）
  - 检测时机：用户点击发布时，后端先进行算法检测
  - 检测结果：通过则进入待审核状态，不通过则直接拒绝并提示用户
- **人工审核**：管理员在后台对通过算法检测的帖子进行二次审核
- **举报机制**：用户举报 → 后台显示 → 管理员审核 → 超过5次自动下架

## 三、功能需求详细说明

### 3.1 微信小程序端（用户端）功能

#### 3.1.1 用户认证模块
- **首次使用（微信一键登录/注册）**
  - 进入小程序后，使用微信授权获取用户信息
  - 使用 `<button open-type="getPhoneNumber">` 获取用户手机号
  - 调用后端接口，传递微信返回的加密数据（code、encryptedData、iv）
  - 后端解密手机号，判断用户是否存在：
    - **如果不存在**：自动注册新用户
      - 自动生成用户ID（UUID 或自增ID）
      - 自动生成昵称（如：用户12345 或 微信昵称）
      - 自动设置头像（使用微信头像或默认头像）
      - 保存手机号
    - **如果已存在**：直接登录
  - 登录/注册成功后，后端返回 token 和用户信息
  - 小程序本地存储 token 和用户基本信息（`wx.setStorageSync`）
- **后续使用（自动登录）**
  - 启动小程序时（`onLaunch`），自动检查本地是否有 token
  - 如果有 token，调用后端接口验证 token 有效性
  - 如果 token 有效，直接进入首页（保持登录状态）
  - 如果 token 无效/过期，跳转到登录页（重新获取手机号）
- **个人资料修改**
  - 在个人中心页面，用户可以修改昵称和头像
  - 修改头像：使用 `wx.chooseImage` 选择图片，上传后更新
  - 修改昵称：输入框修改，调用后端接口更新

#### 3.1.2 帖子发布模块
- **发布入口**：首页底部“发布”按钮
- **发布流程**：
  1. 用户选择帖子类型：**失物招领** 或 **二手物品**
  2. 填写帖子信息：
     - **共同字段**：标题、描述、图片（支持多张，最多 3-5 张）、联系方式（手机号/微信）
     - **失物招领特有**：
       - 丢失/拾到地点
       - 丢失/拾到时间
       - 状态选择：我丢失了 / 我捡到了
     - **二手物品特有**：
       - 物品类别（如：书籍、电子产品、生活用品等）
       - 价格（必填）
       - 新旧程度（可选）
  3. 点击“发布”，调用后端接口提交数据
  4. **后端处理流程**：
     - 接收帖子数据
     - **第一步：算法检测**（内容审核）
       - 对标题、描述进行敏感词检测
       - 如果检测到敏感词，直接返回错误，提示用户修改内容
       - 如果通过检测，继续下一步
     - **第二步：保存到数据库**
       - 帖子状态设置为 `0`（待审核）
       - 保存帖子信息
     - 返回成功，提示“发布成功，等待审核”
  5. 发布成功后，跳转到帖子列表页（但帖子状态为待审核，用户看不到）

#### 3.1.3 帖子浏览模块
- **首页**：
  - 两个 Tab：**失物招领** / **二手物品**
  - 每个 Tab 显示对应类型的帖子列表
  - 列表项显示：封面图（第一张图片）、标题、价格（二手物品）/ 类型（失物招领）、发布时间
- **搜索功能**：
  - 支持关键字搜索（标题、描述）
  - 支持筛选：
    - 失物招领：按地点、物品类型、状态（丢失/拾到）
    - 二手物品：按类别、价格区间
- **帖子详情页**：
  - 显示完整信息（标题、描述、图片轮播、联系方式、发布时间、发布者信息等）
  - **互动功能**：
    - **点赞功能**：
      - 显示点赞数量
      - 点赞按钮（已点赞/未点赞状态切换）
      - 点击点赞，调用后端接口，实时更新点赞数
    - **评论功能**：
      - 显示评论数量
      - 评论列表（显示评论者昵称、头像、评论内容、评论时间）
      - 点击“查看评论”进入评论详情页
      - 评论详情页：显示所有评论，支持发布新评论
  - **底部操作**：
    - 联系发布者（显示联系方式/复制联系方式）
    - **举报按钮**（每个帖子角落都有）
      - 点击举报，弹出举报原因选择（如：虚假信息、违规内容、广告等）
      - 提交举报后，调用后端接口
      - 后端记录举报信息，如果该帖子举报次数达到 5 次，自动将帖子状态改为“已下架”（`status = 3`）

#### 3.1.4 个人中心模块
- **我的发布**：查看我发布的所有帖子（失物招领 + 二手物品）
  - 支持编辑（未审核前可编辑）
  - 支持删除（已发布超过 X 天可删除，或管理员审核拒绝后可删除）
- **个人资料**：查看/修改昵称、头像、联系方式

#### 3.1.5 图片上传功能
- 使用 `wx.chooseImage` 选择图片（相册/拍照）
- 使用 `wx.uploadFile` 上传到后端接口
- 后端接收后保存到服务器，返回图片 URL
- 小程序将 URL 保存到帖子数据中

### 3.2 后台管理端（管理员端）功能

#### 3.2.1 管理员登录
- 管理员账号 + 密码登录
- 登录成功后，使用 Session / Token 保持登录状态

#### 3.2.2 帖子审核管理
- **待审核列表**：显示所有新发布的帖子（默认状态为“待审核”，已通过算法检测）
- **审核操作**：
  - **通过**：帖子状态改为“已通过”（`status = 1`），在小程序端可见
  - **拒绝**：帖子状态改为“已拒绝”（`status = 2`），帖子不显示，可填写拒绝原因
  - **删除**：彻底删除帖子（物理删除或逻辑删除）
- **已审核列表**：查看已审核的帖子（可按状态筛选：已通过、已拒绝、已下架）
- **举报处理列表**：
  - 显示所有被举报的帖子
  - 显示每个帖子的举报次数、举报原因列表
  - 如果举报次数 ≥ 5 次，帖子自动下架（`status = 3`），等待管理员审核
  - 管理员可以：
    - **恢复**：如果审核后认为帖子没问题，可以恢复帖子（`status = 1`）
    - **删除**：如果确实违规，删除帖子
    - **查看详情**：查看帖子内容和所有举报记录

#### 3.2.3 用户管理（可选，功能可简化）
- 查看用户列表
- 查看用户发布的帖子数量
- 封禁/解封用户（可选）

#### 3.2.4 数据统计（可选，功能可简化）
- 总用户数、总帖子数
- 今日新增帖子数
- 失物招领/二手物品数量统计

## 四、数据库设计（核心表结构）

### 4.1 用户表（user）
```sql
CREATE TABLE `user` (
  `id` INT PRIMARY KEY AUTO_INCREMENT,
  `openid` VARCHAR(100) UNIQUE COMMENT '微信openid（唯一标识）',
  `phone` VARCHAR(20) UNIQUE NOT NULL COMMENT '手机号（微信获取）',
  `nickname` VARCHAR(50) DEFAULT '用户' COMMENT '昵称（自动生成，可修改）',
  `avatar_url` VARCHAR(255) COMMENT '头像URL（自动生成，可修改）',
  `wx_nickname` VARCHAR(50) COMMENT '微信昵称',
  `wx_avatar` VARCHAR(255) COMMENT '微信头像',
  `role` TINYINT DEFAULT 1 COMMENT '角色：1-普通用户，2-管理员',
  `status` TINYINT DEFAULT 1 COMMENT '状态：1-正常，0-封禁',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX `idx_phone` (`phone`),
  INDEX `idx_openid` (`openid`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

### 4.2 帖子表（post）
```sql
CREATE TABLE `post` (
  `id` INT PRIMARY KEY AUTO_INCREMENT,
  `user_id` INT NOT NULL COMMENT '发布者ID',
  `type` TINYINT NOT NULL COMMENT '帖子类型：1-失物招领，2-二手物品',
  `title` VARCHAR(200) NOT NULL COMMENT '标题',
  `description` TEXT COMMENT '描述',
  `category` VARCHAR(50) COMMENT '类别（失物招领：物品类型；二手物品：物品类别）',
  `price` DECIMAL(10,2) COMMENT '价格（仅二手物品）',
  `location` VARCHAR(100) COMMENT '地点（仅失物招领）',
  `lost_found_time` DATETIME COMMENT '丢失/拾到时间（仅失物招领）',
  `lost_status` TINYINT COMMENT '失物状态：1-我丢失了，2-我捡到了（仅失物招领）',
  `images` TEXT COMMENT '图片URL列表（JSON格式存储多个URL）',
  `contact` VARCHAR(100) COMMENT '联系方式',
  `status` TINYINT DEFAULT 0 COMMENT '审核状态：0-待审核，1-已通过，2-已拒绝，3-已下架（举报超过5次）',
  `view_count` INT DEFAULT 0 COMMENT '浏览次数',
  `like_count` INT DEFAULT 0 COMMENT '点赞数',
  `comment_count` INT DEFAULT 0 COMMENT '评论数',
  `report_count` INT DEFAULT 0 COMMENT '举报次数',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX `idx_user_id` (`user_id`),
  INDEX `idx_type` (`type`),
  INDEX `idx_status` (`status`),
  INDEX `idx_create_time` (`create_time`),
  INDEX `idx_report_count` (`report_count`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='帖子表';
```

### 4.3 Token表（user_token，用于保持登录状态）
```sql
CREATE TABLE `user_token` (
  `id` INT PRIMARY KEY AUTO_INCREMENT,
  `user_id` INT NOT NULL,
  `token` VARCHAR(255) UNIQUE NOT NULL,
  `expire_time` DATETIME NOT NULL COMMENT '过期时间',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX `idx_user_id` (`user_id`),
  INDEX `idx_token` (`token`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户登录Token表';
```

### 4.4 点赞表（post_like）
```sql
CREATE TABLE `post_like` (
  `id` INT PRIMARY KEY AUTO_INCREMENT,
  `post_id` INT NOT NULL COMMENT '帖子ID',
  `user_id` INT NOT NULL COMMENT '点赞用户ID',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY `uk_post_user` (`post_id`, `user_id`),
  INDEX `idx_post_id` (`post_id`),
  INDEX `idx_user_id` (`user_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='帖子点赞表';
```

### 4.5 评论表（post_comment）
```sql
CREATE TABLE `post_comment` (
  `id` INT PRIMARY KEY AUTO_INCREMENT,
  `post_id` INT NOT NULL COMMENT '帖子ID',
  `user_id` INT NOT NULL COMMENT '评论用户ID',
  `content` TEXT NOT NULL COMMENT '评论内容',
  `parent_id` INT DEFAULT 0 COMMENT '父评论ID（0表示顶级评论，支持回复）',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX `idx_post_id` (`post_id`),
  INDEX `idx_user_id` (`user_id`),
  INDEX `idx_parent_id` (`parent_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='帖子评论表';
```

### 4.6 举报表（post_report）
```sql
CREATE TABLE `post_report` (
  `id` INT PRIMARY KEY AUTO_INCREMENT,
  `post_id` INT NOT NULL COMMENT '被举报的帖子ID',
  `user_id` INT NOT NULL COMMENT '举报用户ID',
  `reason` VARCHAR(100) COMMENT '举报原因（如：虚假信息、违规内容、广告等）',
  `description` TEXT COMMENT '举报描述（可选）',
  `status` TINYINT DEFAULT 0 COMMENT '处理状态：0-待处理，1-已处理（通过），2-已处理（拒绝）',
  `admin_id` INT COMMENT '处理的管理员ID',
  `handle_time` DATETIME COMMENT '处理时间',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX `idx_post_id` (`post_id`),
  INDEX `idx_user_id` (`user_id`),
  INDEX `idx_status` (`status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='帖子举报表';
```

### 4.7 敏感词表（sensitive_word，用于内容审核）
```sql
CREATE TABLE `sensitive_word` (
  `id` INT PRIMARY KEY AUTO_INCREMENT,
  `word` VARCHAR(50) NOT NULL COMMENT '敏感词',
  `level` TINYINT DEFAULT 1 COMMENT '敏感级别：1-一般，2-严重',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY `uk_word` (`word`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='敏感词表';
```

## 五、后端接口设计（RESTful API）

### 5.1 用户认证相关接口

| 接口路径 | 请求方式 | 功能说明 | 请求参数 | 返回示例 |
|---------|---------|---------|---------|---------|
| `/api/user/wxLogin` | POST | 微信登录/注册 | `code, encryptedData, iv`（微信返回的数据） | `{code:200, msg:"登录成功", data:{token:"xxx", user:{id,phone,nickname,avatar_url}}}` |
| `/api/user/verifyToken` | GET | 验证Token | Header: `token` | `{code:200, data:{user:{...}}}` |
| `/api/user/info` | GET | 获取用户信息 | Header: `token` | `{code:200, data:{...}}` |
| `/api/user/updateInfo` | PUT | 更新用户信息（昵称、头像） | Header: `token`, Body: `{nickname, avatar_url}` | `{code:200, msg:"更新成功"}` |

### 5.2 帖子相关接口

| 接口路径 | 请求方式 | 功能说明 | 请求参数 | 返回示例 |
|---------|---------|---------|---------|---------|
| `/api/post/publish` | POST | 发布帖子（先算法检测，通过后保存） | `type, title, description, images, ...` + Header: `token` | `{code:200, msg:"发布成功，等待审核", data:{postId:1}}` 或 `{code:400, msg:"内容包含敏感词"}` |
| `/api/post/list` | GET | 获取帖子列表（只返回已通过的） | `type, page, pageSize, keyword, category, ...` | `{code:200, data:{list:[...], total:100}}` |
| `/api/post/detail/{id}` | GET | 获取帖子详情 | Path: `id` | `{code:200, data:{...}}` |
| `/api/post/myList` | GET | 我的发布列表 | Header: `token`, Query: `page, pageSize` | `{code:200, data:{list:[...]}}` |
| `/api/post/update/{id}` | PUT | 更新帖子 | Path: `id`, Body: `{...}` + Header: `token` | `{code:200, msg:"更新成功"}` |
| `/api/post/delete/{id}` | DELETE | 删除帖子 | Path: `id` + Header: `token` | `{code:200, msg:"删除成功"}` |

### 5.3 点赞相关接口

| 接口路径 | 请求方式 | 功能说明 | 请求参数 | 返回示例 |
|---------|---------|---------|---------|---------|
| `/api/post/like/{id}` | POST | 点赞/取消点赞 | Path: `id` + Header: `token` | `{code:200, data:{liked:true, likeCount:10}}` |
| `/api/post/isLiked/{id}` | GET | 检查是否已点赞 | Path: `id` + Header: `token` | `{code:200, data:{liked:true}}` |

### 5.4 评论相关接口

| 接口路径 | 请求方式 | 功能说明 | 请求参数 | 返回示例 |
|---------|---------|---------|---------|---------|
| `/api/comment/add` | POST | 发布评论 | Body: `{postId, content, parentId}` + Header: `token` | `{code:200, msg:"评论成功", data:{commentId:1}}` |
| `/api/comment/list/{postId}` | GET | 获取帖子评论列表 | Path: `postId`, Query: `page, pageSize` | `{code:200, data:{list:[...], total:50}}` |
| `/api/comment/delete/{id}` | DELETE | 删除评论 | Path: `id` + Header: `token` | `{code:200, msg:"删除成功"}` |

### 5.5 举报相关接口

| 接口路径 | 请求方式 | 功能说明 | 请求参数 | 返回示例 |
|---------|---------|---------|---------|---------|
| `/api/report/add` | POST | 举报帖子 | Body: `{postId, reason, description}` + Header: `token` | `{code:200, msg:"举报成功"}` 或 `{code:200, msg:"举报成功，该帖子已被下架"}`（如果达到5次） |

|---------|---------|---------|---------|---------|
| `/api/upload/image` | POST | 上传图片 | `multipart/form-data: file` + Header: `token` | `{code:200, data:{url:"http://..."}}` |

### 5.7 管理员接口（后台管理端使用）

| 接口路径 | 请求方式 | 功能说明 | 请求参数 | 返回示例 |

## 七、关键技术实现要点

### 7.1 微信登录/注册实现（小程序端 + 后端）
- **小程序端流程**：
  1. 使用 `<button open-type="getPhoneNumber" bindgetphonenumber="getPhoneNumber">` 获取手机号
  2. 调用 `wx.login()` 获取 `code`
  3. 将 `code`、`encryptedData`、`iv` 发送到后端
- **后端流程**：
  1. 使用 `code` 调用微信接口获取 `session_key`
  2. 使用 `session_key` 解密 `encryptedData` 获取手机号
  3. 根据手机号查询用户是否存在：
     - **不存在**：创建新用户
       - 生成用户ID（自增或UUID）
       - 生成默认昵称（如：`用户${随机数}` 或使用微信昵称）
       - 设置默认头像（微信头像或系统默认头像）
     - **已存在**：直接登录
  4. 生成 token，返回给小程序
- **登录状态保持**：
  1. 小程序使用 `wx.setStorageSync('token', token)` 保存到本地
  2. 后续每次请求 API，在 `header` 中携带 `token`
  3. 后端拦截器校验 `token`，有效则放行，无效则返回 401
  4. 小程序启动时（`onLaunch`），检查本地是否有 `token`，有则调用验证接口

### 7.3 内容审核算法实现（敏感词检测）
- **实现方式**：
  1. 创建敏感词表（`sensitive_word`），初始化常用敏感词
  2. 后端实现敏感词检测工具类：
     ```java
     public class SensitiveWordFilter {
         // 加载敏感词库到内存（Trie树或HashSet）
         // 检测方法：遍历文本，查找是否包含敏感词
         public boolean containsSensitiveWord(String text) {
             // 实现敏感词匹配逻辑
         }
     }
     ```
  3. 在帖子发布接口中调用：
     - 检测标题和描述
     - 如果包含敏感词，返回错误提示
     - 如果不包含，继续保存（状态为待审核）
- **优化建议**：
  - 使用 Trie 树（前缀树）提高匹配效率
  - 支持敏感词级别（一般/严重），严重级别直接拒绝
  - 可以扩展为更复杂的 NLP 算法（如情感分析、文本分类等）

### 7.4 点赞功能实现
- **数据库设计**：使用 `post_like` 表记录点赞关系（防止重复点赞）
- **实现流程**：
  1. 用户点击点赞，调用 `/api/post/like/{id}` 接口
  2. 后端检查该用户是否已点赞：
     - 如果未点赞：插入点赞记录，帖子 `like_count` +1
     - 如果已点赞：删除点赞记录，帖子 `like_count` -1
  3. 返回当前点赞状态和点赞数
  4. 小程序更新UI

### 7.5 评论功能实现
- **数据库设计**：使用 `post_comment` 表，支持回复（`parent_id` 字段）
- **实现流程**：
  1. 用户发布评论，调用 `/api/comment/add` 接口
  2. 后端保存评论，帖子 `comment_count` +1
  3. 查看评论列表：调用 `/api/comment/list/{postId}`，支持分页
  4. 评论列表显示：评论者昵称、头像、评论内容、时间

### 7.6 举报功能与自动下架实现
- **举报流程**：
  1. 用户点击举报，选择举报原因，调用 `/api/report/add` 接口
  2. 后端保存举报记录到 `post_report` 表
  3. 更新帖子的 `report_count` 字段（+1）
  4. **关键逻辑**：检查 `report_count` 是否 ≥ 5：
     - 如果 ≥ 5：自动将帖子状态改为 `3`（已下架），返回提示"该帖子已被下架"
     - 如果 < 5：返回"举报成功"
- **管理员处理**：
  1. 后台显示所有被举报的帖子（按举报次数排序）
  2. 显示自动下架的帖子（`status = 3`）
  3. 管理员可以：
     - **恢复**：将状态改为 `1`（已通过），`report_count` 清零
     - **删除**：彻底删除帖子
     - **查看详情**：查看帖子内容和所有举报记录

### 7.7 帖子类型区分
- 在 `post` 表中用 `type` 字段：`1` = 失物招领，`2` = 二手物品
- 小程序端发布时选择类型，提交到后端
- 后端查询时按 `type` 字段筛选

## 九、注意事项

1. **小程序开发准备**：
   - 需要注册微信小程序账号（个人或企业）
   - 在小程序后台配置服务器域名（后端接口域名需要在小程序后台配置白名单）
   - 配置获取手机号权限（需要企业认证，个人开发者可能无法使用，需要准备备选方案）
   - **备选方案**：如果无法获取手机号，可以改为使用微信授权获取用户信息（openid），然后让用户手动输入手机号

2. **图片存储方案**：
   - 方案一：保存到服务器本地目录，配置 Nginx 静态资源访问
   - 方案二：使用对象存储（OSS、COS、七牛云等），更稳定但需要额外配置

3. **安全性**：
   - 微信登录使用官方 API，确保数据安全
   - Token 设置过期时间（如 7 天）
   - 接口参数校验（防止 SQL 注入、XSS 等）
   - 内容审核：算法检测 + 人工审核双重保障
   - 举报机制：防止恶意内容传播

4. **性能优化**（可选）：
   - 帖子列表使用分页，避免一次性加载过多数据
   - 图片使用缩略图，点击详情再加载原图

## 十、总结

本项目采用 **SSM + JSP + Layui + 微信小程序** 的技术栈，实现一个功能完整的校园二手物品与失物招领平台。重点在于：

- **前后端分离**：后端提供统一的 REST 接口，支持小程序和管理后台
- **多端支持**：微信小程序（用户端）+ JSP + Layui 管理后台（管理员端）
- **用户体验**：微信一键登录/注册，自动生成昵称和头像，自动保持登录状态
- **内容安全**：算法检测（敏感词过滤）+ 人工审核 + 用户举报三重保障
- **互动功能**：点赞、评论功能增强用户粘性
- **智能管理**：举报超过5次自动下架，减轻管理员工作量

**开发优先级建议**：
1. **第一阶段**：用户认证（微信登录）、帖子发布与浏览、内容审核算法
2. **第二阶段**：点赞、评论功能
3. **第三阶段**：举报功能、自动下架逻辑
4. **第四阶段**：后台管理端（JSP + Layui）、举报处理
5. **第五阶段**：测试优化、界面美化
